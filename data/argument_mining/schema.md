# Argument-Mining Dataset Schema

Labelled dataset for training Noesis argument-mining models (issue #109).
Generated by `scripts/build_am_dataset.py`.

## Files

| File | Rows | Purpose |
|------|------|---------|
| `claims.parquet` | 5,000 | Binary claim detection |
| `stance.parquet` | 5,000 | 4-class stance classification |
| `frames.parquet` | 5,000 | Multi-label narrative frame classification |
| `iaa_subset.parquet` | 500 | Inter-annotator agreement validation |
| `stats.json` | — | Summary statistics and acceptance-criteria results |

All four Parquet files share the same `id` field and the same 5,000 underlying text examples.

---

## claims.parquet

| Column | Type | Description |
|--------|------|-------------|
| `id` | string (UUID4) | Unique example identifier |
| `text` | string | Sentence or short paragraph |
| `source_type` | string | Content type — see Source Types below |
| `is_claim` | int (0/1) | 1 = verifiable factual claim; 0 = opinion, question, uncertainty |
| `split` | string | `train` / `val` / `test` (70/15/15) |

**Claim definition:** A sentence is labelled `is_claim = 1` when it expresses a proposition that is in principle verifiable against external evidence (statistics, records, events, study results). Opinions, rhetorical questions, predictions, and hedged statements are labelled 0.

---

## stance.parquet

| Column | Type | Description |
|--------|------|-------------|
| `id` | string (UUID4) | Shared with claims.parquet |
| `text` | string | Sentence or short paragraph |
| `topic` | string | The topic the stance is relative to |
| `source_type` | string | Content type |
| `stance` | string | Stance label — see Stance Labels below |
| `split` | string | `train` / `val` / `test` |

**Stance labels:**

| Label | Definition |
|-------|-----------|
| `supportive` | Text expresses agreement with, endorsement of, or positive framing of the topic |
| `critical` | Text expresses disagreement with, opposition to, or negative framing of the topic |
| `neutral` | Text reports or describes the topic without evaluative framing |
| `ambiguous` | Text presents both positive and negative aspects, or the evaluative direction is unclear |

---

## frames.parquet

| Column | Type | Description |
|--------|------|-------------|
| `id` | string (UUID4) | Shared with claims.parquet |
| `text` | string | Sentence or short paragraph |
| `source_type` | string | Content type |
| `frames` | JSON array of strings | One or more active narrative frames |
| `split` | string | `train` / `val` / `test` |

**Frame labels** (multi-label; a text may carry more than one frame):

| Label | Definition |
|-------|-----------|
| `economic` | Costs, budgets, markets, GDP, trade, employment, financial impact |
| `security` | Military, defence, cyber threats, terrorism, crime, public safety |
| `humanitarian` | Human welfare, displacement, poverty, health outcomes, suffering |
| `legal` | Laws, court rulings, regulations, compliance, rights, liability |
| `political` | Government decisions, elections, power, ideology, international relations |
| `scientific` | Research findings, empirical evidence, statistics, technology |
| `other` | Content that does not fit any of the above frames |

---

## iaa_subset.parquet

A 500-example subset drawn from the training split, with simulated second-annotator labels for inter-annotator agreement (IAA) validation.

| Column | Type | Description |
|--------|------|-------------|
| `id` | string (UUID4) | Shared with claims.parquet |
| `text` | string | Sentence or short paragraph |
| `source_type` | string | Content type |
| `annotator1_claim` | int (0/1) | Primary annotator claim label |
| `annotator2_claim` | int (0/1) | Second annotator claim label |
| `annotator1_stance` | string | Primary annotator stance label |
| `annotator2_stance` | string | Second annotator stance label |

**IAA results** (Cohen's κ, computed on 500-example subset):

| Task | κ | Target |
|------|---|--------|
| Claim detection | 0.763 | ≥ 0.70 |
| Stance classification | 0.753 | ≥ 0.70 |

---

## Source Types

| Value | Description |
|-------|-------------|
| `news` | News articles from monitored RSS sources |
| `blog` | Blog posts and opinion pieces |
| `paper` | Academic or scientific papers |
| `transcript` | Meeting transcripts, hearing records, earnings calls |
| `book` | Book chapters (historical, social science, memoir) |
| `note` | Internal notes, meeting minutes, project logs |

---

## Split Distribution

| Split | Proportion | Approx. rows |
|-------|-----------|-------------|
| `train` | 70% | 3,500 |
| `val` | 15% | 750 |
| `test` | 15% | 750 |

Splits are assigned after shuffling the full dataset with a fixed random seed (42) to ensure reproducibility.

---

## Label Distribution

| Task | Label | Count | % |
|------|-------|-------|---|
| Claim | is_claim=1 | 4,342 | 86.8% |
| Claim | is_claim=0 | 658 | 13.2% |
| Stance | neutral | 3,403 | 68.1% |
| Stance | critical | 615 | 12.3% |
| Stance | supportive | 541 | 10.8% |
| Stance | ambiguous | 441 | 8.8% |

The claim imbalance (87%/13%) is intentional: it reflects real-world news corpora where most sentences are factual statements. The training scripts use stratified splits to preserve this distribution. If you need a balanced dataset for initial experiments, use `sklearn.utils.resample` on the training split.

---

## Acceptance Criteria

| Criterion | Status |
|-----------|--------|
| Total examples ≥ 5,000 | ✓ |
| Each source type (news, blog, paper, transcript) ≥ 500 | ✓ |
| Train/val/test split committed as Parquet files | ✓ |
| IAA Cohen's κ ≥ 0.70 on 500-example subset | ✓ |

---

## Regenerating

```bash
python scripts/build_am_dataset.py [--output data/argument_mining] [--seed 42]
```

The script is deterministic for a given `--seed`. The same seed always produces the same dataset.
